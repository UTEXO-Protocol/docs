The Utexo Bridge is a non-custodial, atomic cross-chain bridge that moves USDT between EVM-based networks (Ethereum, Tron) and the Bitcoin RGB layer. Bridged USDT is represented on Bitcoin as native RGB USDT (TUSDT) — a real on-chain asset that can be transferred or exchanged against BTC with on-chain settlement guarantees, without relying on centralised exchanges, wrapped assets or custodial intermediaries.

## How It Works

The Bridge is composed of modular, stateless components that together coordinate a fully atomic transfer:

| Component | Role |
| --- | --- |
| **Bridge orchestrator** | Tracks all in-flight transfers, routes events between connectors, and drives transfers to completion or cancellation |
| **Connectors** | Chain-specific gRPC services that monitor smart contracts for `FundsIn` events and issue `FundsOut` transactions on the destination chain |
| **Enclave Signer** | Cryptographic microservice running inside an AWS Nitro Enclave; derives and holds private keys in a Trusted Execution Environment (TEE) so they never leave the hardware boundary |
| **REST Gateway** | Public API used by the web UI and external integrations |

### Bridge Orchestrator

The Bridge orchestrator is the central coordinator. It tracks all in-flight transfers in a persistent database, processes `FundsIn` events reported by Connectors, and instructs the appropriate destination Connector to issue a `FundsOut` transaction. If a transfer cannot be completed, the orchestrator drives it to safe cancellation.

### Connectors

Each supported chain has a dedicated Connector — a stateless gRPC service that watches the chain for `FundsIn` smart contract events and issues `FundsOut` transactions on the destination chain. Connectors do not hold keys or funds. All signing is delegated to the Enclave Signer.

### Enclave Signer

The Enclave Signer (`utexo-bridge-enclave-signer`) is a cryptographic microservice running inside an **AWS Nitro Enclave** — an isolated compute environment with no persistent storage, no shell access, and network access limited to a vsock proxy allowlisted to an Esplora indexer.

Key properties:

- **Keys never leave the TEE.** BIP-39 key generation and all signing operations (EIP-712 for EVM, PSBT Taproot Schnorr for Bitcoin) happen inside the enclave.
- **In-enclave RGB validation.** Every RGB consignment is validated against an Esplora indexer before signing, cross-checking amounts against calldata and commission.
- **Memory safety.** Secrets are zeroized on drop; the codebase enforces `#![deny(unsafe_code)]`.

## Transfer Flow

The following steps describe a complete atomic transfer:

1. User deposits funds on the source chain (`FundsIn`).
2. The source Connector detects the on-chain event and notifies the Bridge orchestrator.
3. The Bridge orchestrator instructs the destination Connector to release funds (`FundsOut`).
4. The Enclave Signer signs the `FundsOut` transaction inside the TEE before broadcast.
5. Transfer is marked complete once finality is confirmed.

**Example — Ethereum → Bitcoin RGB layer:**

1. User submits a `FundsIn` transaction on the Ethereum smart contract.
2. The Ethereum Connector detects the event and notifies the orchestrator.
3. The orchestrator instructs the Bitcoin Connector to prepare a `FundsOut` PSBT.
4. The Enclave Signer validates the RGB consignment against Esplora and signs the PSBT using Taproot Schnorr.
5. The signed transaction is broadcast to Bitcoin. Once confirmed, TUSDT appears in the user's RGB wallet.

## Supported Networks

| Network | Type | Direction |
| --- | --- | --- |
| Ethereum | EVM (secp256k1) | Inbound & Outbound |
| Tron | EVM-compatible | Inbound & Outbound |
| Bitcoin (RGB layer) | Taproot / Schnorr | Inbound & Outbound |

## Supported Assets

| Asset | Source Network | Representation on Bitcoin |
| --- | --- | --- |
| USDT | Ethereum, Tron | RGB USDT (TUSDT) |

## Security

The Enclave Signer is the security boundary of the Bridge. It runs inside an AWS Nitro Enclave, ensuring that the private keys used to sign EVM and Bitcoin transactions are never accessible outside the hardware boundary — not to the Bridge orchestrator, not to infrastructure operators, and not to any external service.

The Esplora indexer connection via vsock proxy is the only network interface allowed from within the enclave. This limits the attack surface to a single, auditable read-only data source used exclusively for RGB consignment validation.
